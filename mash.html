Hurricane Visualization

This project was an excursion into a purely visual form of 3D modeling. The project came about as a desire to utilize the new tool pack that was packaged into the Maya 2017 version update called MASH. The powerful collection of features included a more functional version of Maya's tool XGen. XGen is used typically for procedurally generated features and can place and manipulate objects according to given functions. I had previously attempted to produce this project's effect using XGen with unsatisfactory results. While the process worked with XGen, the issue arose when trying to render the scene that the objects that were created did not respond well with the scene's lighting. Most of these issues were solved by using MASH, although object's materials got replaced by a default material due since the colors were being procedurally manipulated, leading to a lackluster sheen.

The scene setup was fairly basic. It began with a single grey cube. Then MASH was used to turn this one cube into a grid of 500 by 500 of these cubes, 250,000 total. Each of these cubes acted as a single pixel, each being able to be manipulated through a script, like a low definition screen. Once I had the grid I chose a photo that would translate well into a 3D scene. Once I had my photo I converted it into greyscale and applied a pixilation filter with a resolution that would give a 500 by 500 result that matched my grid of cubes. This photo was applied as the cube height map, multiplying the cubes' heights' by the greyscale value of the pixel on top of each cube.


Finally, in order to give the cubes colors, I used a process extremely similar to how I made the height map. Using the same original image I again applied the pixilation filter. However, this time I kept the original colors instead of using the greyscale values. This image was applied as a texture map and the colors were projected onto the cube underneath each pixel.


Satisfied with the result, after conceding that I wouldn't be able to give the cubes a glossy attribute in their texture, I set up the scene for a final render. This process encompassed positioning two plane lights and an area light to give the scene a natural look. These individual lights' positions can easily be seen in the photos above. The final step in the process was placing the camera at the best angle in order to not see the edges of the scene's plane and allow for some soft shadows to be picked up. The scene was then rendered in a 4k resolution square, taking about 20 minutes.
